Go by Example: Slices

Goにおいて、_スライス_は連続した要素を扱う処理において 配列よりも強力なインターフェースを提供する、とても重要なデータ型です。

 
package main

配列とは異なり、スライスは、要素の数ではなく、要素に含まれる要素で型宣言がされます。 長さがゼロでない空のスライスを作成するには、組み込みの makeを使います。 ここでは、長さ 3 '（最初はゼロ）の string`スライスを作成します。

 
    s := make([]string, 3)
    fmt.Println("emp:", s)

配列のように値をセットしたり、取得したりすることができます。

 
    s[0] = "a"
    s[1] = "b"
    s[2] = "c"
    fmt.Println("set:", s)
    fmt.Println("get:", s[2])

lenでスライスの長さを取得することができます。

 
    fmt.Println("len:", len(s))

これら基本的な操作に加えて、スライスは配列よりもリッチな機能をいくつかサポートしています。 1つは appendで、1つ以上の新しい値を含むスライスを返します。 新しいスライスの値を取得するには、 appendからの戻り値を取得する必要があることにご注意してください。

 
    s = append(s, "d")
    s = append(s, "e", "f")
    fmt.Println("apd:", s)

スライスは copyすることもできます。 ここではsと同じ長さの空のスライスcを作成し、sからcにコピーします。

 
    c := make([]string, len(s))
    copy(c, s)
    fmt.Println("cpy:", c)

スライスは、slice[low:high]構文でsliceする(断片を抜き出す) オペレーターをサポートします。 例えば、この例では要素s[2]、s[3]とs[4]をスライスします。

 
    l := s[2:5]
    fmt.Println("sl1:", l)

これは s[5]までの要素（s[5]は含まない）をスライスします。

 
    l = s[:5]
    fmt.Println("sl2:", l)

これはs[2]からの要素（s[2]を含む）をスライスします。

 
    l = s[2:]
    fmt.Println("sl3:", l)

一行でスライスの変数を宣言して初期化することもできます。

 
    t := []string{"g", "h", "i"}
    fmt.Println("dcl:", t)

スライスは、多次元データ構造に構成することができます。 内部スライスの長さは、多次元配列とは異なり、可変です。

 
    twoD := make([][]int, 3)
    for i := 0; i < 3; i++ {
        innerLen := i + 1
        twoD[i] = make([]int, innerLen)
        for j := 0; j < innerLen; j++ {
            twoD[i][j] = i + j
        }
    }
    fmt.Println("2d: ", twoD)
}

スライスは配列とは異なる型ですが、fmt.Printlnでは同じようにレンダリング されることにご注意ください。
