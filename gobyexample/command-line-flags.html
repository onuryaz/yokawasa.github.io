Go by Example: Command-Line Flags

コマンドラインフラグはコマンドラインプログラムのオプションを指定する一般的な方法です。 たとえば、 wc -lでは-lがコマンドラインフラグになります。

 
package main

Goは、基本的なコマンドラインフラグ解析をサポートするflagパッケージを提供しています。 このパッケージを使用して、サンプルのコマンドラインプログラムを実装します。

 
import "flag"
import "fmt"

func main() {

基本フラグの宣言は、文字列、整数、およびブールオプションで使用できます。 ここでは、文字列フラグwordをデフォルト値"foo"とフラグの簡単な説明をつけて宣言します。 この flag.String関数は、文字列ポインタ（文字列値ではありません）を返します。 以下、このポインタを使用する方法を見ていきます。

 
	wordPtr := flag.String("word", "foo", "a string")

ここではwordフラグに似たアプローチでnumbとforkフラグを宣言しています。

 
	numbPtr := flag.Int("numb", 42, "an int")
	boolPtr := flag.Bool("fork", false, "a bool")

プログラムの別の場所で宣言されている既存のvarを使用してオプションを宣言することもできます。 フラグを宣言している関数にポインタを渡す必要があることに注意してください。

 
	var svar string
	flag.StringVar(&svar, "svar", "bar", "a string var")

すべてのフラグが宣言されたら、flag.Parse()を呼び出してコマンドラインの解析を実行します。

 
	flag.Parse()

ここでは、解析されたオプションと後続の位置引数をダンプします。 実際のオプション値を取得するにはポインタを逆参照する必要があることにご注意ください（例、* wordPtr）。

 
	fmt.Println("word:", *wordPtr)
	fmt.Println("numb:", *numbPtr)
	fmt.Println("fork:", *boolPtr)
	fmt.Println("svar:", svar)
	fmt.Println("tail:", flag.Args())
}

コマンドラインフラグを試すには、直接バイナリをビルドして実行させてみてするのが一番です。

 
$ go build command-line-flags.go

最初にすべてのフラグに値を与えて、ビルドされたプログラムを試してみてください。

 
$ ./command-line-flags -word=opt -numb=7 -fork -svar=flag
word: opt
numb: 7
fork: true
svar: flag
tail: []

フラグを省略すると、自動的にデフォルト値が使用されます。

 
$ ./command-line-flags -word=opt
word: opt
numb: 42
fork: false
svar: bar
tail: []

後続の位置引数は、任意のフラグの後に指定できます。

 
$ ./command-line-flags -word=opt a1 a2 a3
word: opt
...
tail: [a1 a2 a3]

flagパッケージは、すべてのフラグをフラグなし引数の前に指定することを要求します（そうでなければ、フラグはフラグなしの引数として解釈されます）。

 
$ ./command-line-flags -word=opt a1 a2 a3 -numb=7
word: opt
numb: 42
fork: false
svar: bar
tail: [a1 a2 a3 -numb=7]

-hまたは--helpでコマンドラインプログラムのヘルプテキストが自動生成されます。

 
$ ./command-line-flags -h
Usage of ./command-line-flags:
  -fork=false: a bool
  -numb=42: an int
  -svar="bar": a string var
  -word="foo": a string

flagパッケージに指定されていないフラグを指定すると、プログラムはエラーメッセージを出力し、ヘルプテキストを再度表示します。

 
$ ./command-line-flags -wat
flag provided but not defined: -wat
Usage of ./command-line-flags:
...
